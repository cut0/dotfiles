# コード品質ルール

## 全言語共通ルール

- linter/formatter に従う: プロジェクトの linter/formatter 設定に従う

## 必須制約事項

- 完了前にテストを実行: タスク完了前にテストを実行
- リントがパスすることを確認: lint エラーがないことを確認
- コメントアウトされたコード禁止: 不要なコードは削除
- TODO コメント禁止: TODO は残さず解決
- 明示的なエラーハンドリング: エラーは適切に処理

## 作業ディレクトリの報告

- 編集作業前に報告: 編集を始める前にカレントディレクトリを報告
- ディレクトリ移動後に報告: cd 後は現在のパスを報告
- pwd コマンドを使用: パスを明示する際は pwd を使用
