# Writing スタイルガイド

## 避けるべき表現

- Markdown の強調記法（`**`、`*`）を多用しない
- 絵文字を使わない（明示的に指示された場合を除く）
- 「素晴らしい」「完璧」などの過剰な賛辞を避ける
- 「もちろん」「確かに」などの冗長な前置きを避ける
- 箇条書きの各項目を太字で始めない

## 推奨されるスタイル

- 簡潔で直接的な表現を使う
- 事実に基づいた客観的な記述をする
- 技術的な正確さを優先する
- 必要な情報を過不足なく伝える

## フォーマット

- 見出しは `#` を使用
- リストは `-` を使用
- コードは ``` で囲む
- 強調が必要な場合は `バッククォート` を使用
